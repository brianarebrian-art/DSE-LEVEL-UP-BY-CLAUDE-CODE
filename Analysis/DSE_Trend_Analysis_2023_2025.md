## 0. 全局最大變革：通識科（LS）→ 公民與社會發展科（CSD）

這是 2023–2025 期間 HKDSE 最重大、且已由官方確認的結構性改變：

- 通識教育科（Liberal Studies）**最後一屆公開試為 2023 年**；公民與社會發展科（Citizenship and Social Development, CSD／公社）**首屆公開試為 2024 年**。
- **評核方式徹底改變**：CSD 為必修核心科，設**單一份約 2 小時的筆試，內含三條資料回應題（data-response）**，成績**只分「達標／不達標」（Attained／Unattained）兩級**，不再用 1–5 或星級。
- 課程收窄為三大主題：香港在「一國兩制」下的發展、國家自改革開放以來的發展、當代世界的互聯與互依；並含**內地考察**（不列入公開評核）。
- **對備試的直接影響**：由「高分拉開差距」轉為「穩守達標」；答題重心是**準確運用資料 + 概念定位**，而非長篇論證堆砌。2026 一屆並見**更著重考生自身外部知識（external knowledge）**的資料題傾向。

> 對 DSE LEVEL UP 的啟示：公社科**缺乏深層次拉分題**（達標制使然），平台價值應放在「**確保達標**的資料題框架訓練」，而非製造「拔尖」錯覺。

## 1. 三維度分析（分科）

每科按三個維度展開：**① 考點演化**（新題型／跨章節綜合）、**② 失分陷阱**（常見失分 + 評卷關鍵字）、**③ 考評變革**。

### 1.1 中文（Chinese Language）

**① 考點演化**
- **文言與白話並重、重視「篇章互涉」**：卷一閱讀長年兼收文言（classical）與白話（vernacular）篇章；**指定文言篇章（12 篇範文）**的考核，趨向**跨篇concept 綜合**（例如同一主題／論證手法在不同篇章的比較），而非單篇背誦默寫。
- **綜合能力卷**（聆聽＋整合）持續考核「**資料重整＋見解論證**」的實用寫作能力。

**② 失分陷阱（評卷關鍵字層面，屬通則）**
- **過度推論（over-inference）**：閱讀理解答案「講多過篇章所給」，是最典型失分。
- **審題不清（misreading the prompt）**：忽略「最」「除了」「不」等限定詞。
- **文言字詞理解**：一字多義（如「兵」「走」「愛」古今義）判斷錯誤。
- **寫作離題／文體錯配**：實用文體（如書信、建議書）的**語境與格式**未達要求。

**③ 考評變革**
- 口試（卷四）自 2021 年起取消後**維持不設**，備試重心進一步落在**閱讀深度 + 寫作 + 聆聽整合**。

**頻率（質性）**：跨篇綜合題 = **High**；文言字詞精讀 = **High**；純默寫背誦 = **Low（已淡化）**。

---

### 1.2 英文（English Language）

**① 考點演化**
- **Reading（卷一）** 難度分部（Part A/B）持續；重視**跨段落推論、指代與銜接（reference & cohesion）**。
- **Integrated Skills（卷三，聽＋讀寫整合）**：核心能力是**從 Data File 抽取、重組、轉化**資訊，並以**正確語域**輸出。

**② 失分陷阱（通則）**
- **盲抄（blind copying）**：把 Data File 原句照搬入作答，是卷三最大扣分位——考核的是**改寫與整合（paraphrase & integrate）**，非複製。
- **語氣與語域錯配（Tone & Register mismatch）**：卷二寫作，正式（formal）／非正式（informal）語域用錯，或體裁慣例（genre conventions）不符。
- **Task fulfilment 不足**：漏答 prompt 其中一個 bullet / 要求。

**③ 考評變革**
- 卷別結構穩定（Reading／Writing／Listening & Integrated／Speaking）；**評核重心持續偏向「真實語境的語言運用」**多於機械語法。

**頻率（質性）**：Data File 整合改寫 = **High**；Tone & Register 判斷 = **High**；純文法選擇 = **Med→Low**。

---

### 1.3 數學（必修 Compulsory Part）

**① 考點演化**
- **跨課題綜合（cross-topic）**：一題內結合多個課題（如：函數＋二次方程＋圖形變換；三角＋測量；概率＋統計），是拉分關鍵。
- **應用情境化**：把純運算包裝成情境題（financial / measurement context），考**建模 + 讀題**。

**② 失分陷阱（通則）**
- **單位陷阱（unit traps）**：cm↔m、g↔kg、面積／體積單位換算漏轉。
- **運算粗心（careless calculation）**：識做但計錯；**不展示步驟（no working shown）** 導致 method mark 全失。
- **近似與有效數字**：最後一步 rounding 過早或位數錯。

**③ 考評變革／備試技術**
- **計算機程式（Casio fx-50FH II／3650P）** 的善用（如解方程、統計、複利），是**合規的效率技術**，平台應提供 program 教學（術語：計算機神技）。

**頻率（質性）**：跨課題綜合題 = **High**；單位／粗心失分 = **High（最普遍）**；純單一課題直算 = **Med**。

---

### 1.4 經濟（Economics，選修）

> **術語紅線（硬守）**：Public Good 一律譯「**共用品**」（**禁**「公共財」）；Entrepreneurship 一律「**企業家職能**」（**禁**單用「企業」）；**嚴禁大學超綱概念**（如「收入彈性」income elasticity 之類越出 DSE 課綱者）。

**① 考點演化**
- **概念鏈條題**：由定義 → 前提 → 推論的**因果鏈**（如外部性 externality → 市場失效 → 政府干預），單靠背定義不足。
- **圖表分析**：供求、彈性、成本曲線的**移動 vs 沿線移動**辨析。

**② 失分陷阱（通則）**
- **概念盲區（conceptual blind spot）**：忽略定理的**前提條件**（如完全競爭假設）。
- **術語誤用**：把「共用品」「共有資源」「準共用品」混淆；把彈性大小與斜率直接等同。
- **越綱作答**：搬用大學概念，反而失分或偏離課綱。

**③ 考評變革**
- 卷一（MC）＋ 卷二（結構／論述）結構穩定；**重視以課綱框架準確作答**，而非堆術語。

**頻率（質性）**：概念鏈條 + 圖表辨析 = **High**；術語精準 = **High**；純定義背誦 = **Low**。

---

### 1.5 公民與社會發展科（CSD／公社）

**① 考點演化**
- **三條資料回應題（data-response）** 為唯一題型；核心是**準確解讀資料 + 定位主題概念 + 適度連結外部知識**。
- 2026 一屆傾向**更多需要考生自身外部知識**的資料題（非純資料內作答）。

**② 失分陷阱（通則）**
- **只抄資料、不作分析**：資料回應題若只複述圖表數字而無詮釋，得分有限。
- **主題定位錯誤**：把資料錯配到不相關主題框架。

**③ 考評變革**
- **達標／不達標兩級制**：策略由「爭高分」轉為「**穩守達標**」；平台應提供**達標導向**的資料題框架與檢查清單，而非虛構「拉分」。

**頻率（質性）**：資料詮釋 + 外部知識連結 = **High**；純資料複述 = **Low（易失分）**。

## 2. `[Verify Checkpoint 2]` — Top 5 核心趨勢（終端摘要）

1. **達標制取代拉分（CSD）**：通識 2023 末屆 → 公社 2024 首屆，只評達標／不達標；備試重心＝穩守，不是拔尖。
2. **跨章節綜合題成主流拉分位**（數學、中文、經濟）：單一課題直算＋純背誦持續淡化。
3. **「改寫整合」勝「照抄」**（英文卷三、中文閱讀）：盲抄 Data File／過度推論＝最典型失分。
4. **細節性失分主導總失分**：單位換算、運算粗心、不展示步驟、審題漏字——高頻但可訓練。
5. **術語精準是經濟科生死線**：共用品／企業家職能等課綱用語，錯一個扣一個；越綱概念反致失分。

## 3. 對「DSE Level Up」題目庫的合規啟示（引向階段三）

- **只用 Archetype Masking 原創題**：訓練上述**同一底層技能**（跨章節綜合、改寫整合、單位陷阱、術語精準、資料詮釋），但**數字／情境／人名全部原創**，**不高仿真複製**任何真卷。
- **一律入 `drafts/`**：機器產出 → `review-drafts.mjs` → 真人逐題審 → `promote-drafts.mjs`，**永不自動入庫**（生產紀律）。
- **每題掛元數據**：`ref_years`、近三年頻率（High/Med/Low）、陷阱等級（1–5）、雙語對照——供人手審核與教學對標，**頻率標籤沿用本報告的質性判斷**，不冒充精確統計。
- **過守閘**：`term-guard.mjs`（術語紅線）、`validate-banks.mjs`（格式）、`i18n-guard`（雙語一致）。
