# 支援工單模板與流程（Jack / 客戶體驗）

> 用於 email 支援 + IG Group 精選動態置頂。承諾：48 小時內首次回應。

## 工單分類

| 類別 | 例子 | 目標處理時間 |
|------|------|-------------|
| 🔴 題目報錯 | 答案錯、解析錯、超綱、錯字 | 48 小時內核實，屬實即修正 |
| 🟠 功能故障 | 載入失敗、進度唔見、登入問題 | 48 小時內回覆 + 修復排期 |
| 🟡 功能建議 | 想要新科目、新功能 | 每週彙整一次入 roadmap |
| 🟢 一般查詢 | 用法、合作、私隱 | 48 小時內回覆 |

## 報錯模板（貼俾用戶填）

```
【題目報錯】
科目：
題目編號（結果頁可見）／或貼題目首句：
問題類型：答案錯 / 解析錯 / 超綱 / 錯字 / 其他
你認為正確嘅係：
（有來源更好，例如課本章節）
```

## 內部處理流程

1. **接收**：標記類別 + 回覆確認（「收到，48 小時內覆你」）
2. **核實**（題目報錯）：對照課程大綱／教科書 → 屬實：修正 + 過 term-guard/validate 兩個 gate + build → 唔屬實：禮貌解釋點解題目正確（引用概念，唔好淨係話「你錯」）
3. **回報**：話返俾報錯者知結果，並喺 IG Group 公開多謝（鼓勵報錯文化）
4. **升級**：涉及私隱／法律／大規模故障 → 即時通知創辦人

## 回覆語氣指引

- 學長學姐式：「多謝你幫成個平台捉蟲 🙏」
- 禁止機械客服腔（「親愛的用戶您好」❌）
- 用戶報錯但其實題目冇錯：當成一次免費教學機會，認真解釋
